# TradeBook 保有銘柄カード 設計

- 日付: 2026-05-31
- 対象: 公開リポジトリ（TradeBook 本体）

## 背景と目的

これまで「今いくつ・いくらで持っているか」を常時確認できる場所がなかった
（売却フォームで「売」を選んだときに保有ピッカーが一時的に見えるだけ）。
保有状況（ポジション）を一覧できる専用カードを追加する。

なお「売りに対応する買いの整合性」は `calcRealized` の warnings（保有数量を超える
売却の検知）＋警告バナーで既にカバーしているため、本対応では扱わない。

## スコープ

- 表示は **銘柄 / 保有数 / 平均取得単価** の3列。
- 含み損益（損益列）は価格連携が未導入のため**今は出さない**（導入時に列追加）。
- データは既存の `calcRealized(...).holdings` をそのまま使う（新規計算ロジックなし）。

## 画面

「損益サマリー」カードの直後（トレード成績KPIの前）に **「保有銘柄」** カードを追加する。

### テーブル（3列）

| 列 | 内容 | 出所 |
|---|---|---|
| 銘柄 | 社名 ＋ コード | `codeToName(code)` ＋ code |
| 保有数 | 残株数 | `holdings[code].quantity` |
| 平均取得単価 | 円（手数料込み） | `holdings[code].cost / quantity` |

- 並び順: 取得総額（`cost`）の大きい順。
- 0件時: 「保有中の銘柄はありません」。
- 保有数は平均法の残株数をそのまま表示（単元100株前提の丸めはしない。
  保有超過売却後の状態も正しく見えるようにするため）。

## データフロー

```
store.getTrades()
  → withMatsuiFees()        手数料を取得原価に反映
  → calcRealized().holdings  { code: { quantity, cost } }
  → renderHoldings()         quantity>0 のみ・cost降順で描画
```

売却フォームの保有ピッカーと同じ `holdings` を使うため、両者の数値は必ず一致する。

## 実装対象

- `index.html`: 保有銘柄カードのDOM（損益サマリー直後）。
- `js/app.js`: `renderHoldings()` を追加し `renderAll()` に組込。
  ユーザー由来テキストは既存の `esc()` を通す。
- `css/style.css`: テーブル装飾（既存スタイルを流用）。
- `sw.js`: キャッシュ版数 v8 → v9。

## テスト

表示は既存の `holdings` を使うだけで `pnl.js` に新規ロジックを追加しないため、
単体テストの追加は行わない（既存30件の非回帰のみ確認）。実画面はヘッドレス
ブラウザのスクリーンショットで目視確認する。

## 非対象（将来）

- 含み損益・現在値（価格連携の導入時に損益列を追加）。
- 単元未満（端株）の特別表示。
