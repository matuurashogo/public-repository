# 保有銘柄の利確目標（ボラ連動）設計

- 日付: 2026-06-16
- 関連ADR: TBK-0010（データ契約）
- 状態: 設計合意済み（実装着手）

## 背景・課題

保有カードは含み損益（％）を出すだけで「いつ売るか」の目安が無い。実運用は**+5%固定の利確**で回しているが、

1. 伸びる銘柄の利を取りこぼす（利益最大化の妨げ）
2. 銘柄ごとの「動きの大きさ」を考慮できない（穏やかな株も荒い株も一律5%）

という課題がある。買い時は TBK-0006（買い時ボード）で数値化済みなので、その**売り版**を保有カードに足す。

## 検討したアプローチと却下理由

| 案 | 却下/採用 | 理由 |
|---|---|---|
| チャンデリア・トレール（最高値から−Nσ） | 却下 | 最高値追従＝実質リアルタイム前提。終値・約20分遅延データでは機能しない |
| 本物のATR（高値−安値ベース） | 却下 | jquants-data は調整後終値のみで高安が無い（ExitLab/VolDip も同理由でσ代用） |
| 平均日次変化幅（\|前日比\|の平均） | 不採用 | σ20とほぼ比例（正規近似で mean\|r\|≈0.8σ）。ボラ定義を2つに増やすだけで利得なし |
| **σ20連動の静的利確目標** | **採用** | エントリー時に目標価格が確定＝リアルタイム不要。既存σ定義に統一できる |

## 採用設計

### 利確幅（終値ベース）
```
利確幅 width = min( cap, max( floor, k × σ20 ) )
利確目標価格 = 平均取得単価 × (1 + width)
既定: k = 2.5 / floor = 5% / cap = 15% / σ窓 = 20日
```
- σ20＝2%の“普通”の銘柄 → 2.5×2%＝**5%**＝現行と一致（移行が自然）。
- 穏やか（σ小）→ floorの5%死守。荒い（σ大）→ 目標を引き上げ、capの15%で暴走防止。
- σ20 = 直近20日の日次単純リターンの標準偏差（標本・ddof=1）。VolDip/ExitLab と同定義。

### 計算場所
- **σ20はサーバ（GitHub Actions）で日次生成**。銘柄ごとの定数なので1銘柄1数値だけ配信（`data/volatility.json`）。
- **利確目標価格はブラウザで合成**。取得単価はユーザーの TradeBook にしかないため。

### 表示（保有カードに1列追加「利確目標」）
- メイン: 利確目標価格（例 `5,400円`）
- サブ: 未到達 `あと +4.8%` ／ 到達 `🎯到達`（緑ハイライト）
- 根拠（小さく/title）: `目標 +8.0%（σ連動）`
- 含み損益列はそのまま。「今 +3.2% ／ 目標 +8.0%」と並びで判断できる。

### 性質
- **終値ベース・表示専用**。到達判定・含み益％は終値ベース。場中（TBK-0008）は現在値の参考表示のみで、利確判定の根拠にはしない。
- σが無い銘柄（履歴不足）は利確目標を `—`（劣化動作）。JSON が無くてもカードは従来どおり出す。

## テスト方針
- Python: `compute_sigma20()` の値・履歴不足時 None を検証。
- JS: 利確幅 clamp（floor/cap境界）・目標価格・状態（到達/あと%）・σ欠損時の劣化を検証。
