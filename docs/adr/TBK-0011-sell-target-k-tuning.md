# [TBK-0011] 利確目標の係数 k を 2.0 に改定（ボラ連動・volatility.json）

| **Date** | 2026-06-16 |

## ❓ コンテキスト（背景と課題）

TBK-0010 で導入した「ボラ連動の利確目標」を実データで確認したところ、高ボラ銘柄で利確目標が
遠すぎると感じられた。例: 芝浦メカトロニクス（6590）は σ20＝5.21% のため、`k=2.5` だと
利確幅 = 2.5×5.21% ≒ **13.0%** となり、取得単価約4,780円に対し目標が約5,400円まで離れていた。

これは「荒い銘柄ほど目標を広げる」設計どおりの挙動だが、運用感覚として利確が遠すぎたため、
係数 `k` を一段下げて全体的に早めの利確寄りに調整する。**データ契約（JSONスキーマ）は不変**で、
変更するのは既定パラメータ `k` の値のみ。

## 💡 決定事項（Decision）

1. **利確幅の係数 `k` を 2.5 → 2.0 に改定する**。式・スキーマ・他パラメータは TBK-0010 のまま:
   ```
   利確幅 width = min( cap, max( floor, k × σ20 ) )
   利確目標価格 = 平均取得単価 × (1 + width)
   既定: k = 2.0 / floor = 0.05 / cap = 0.15 / σ窓 = 20日
   ```
   - 改定後の例: 6590（σ5.21%）→ 2.0×5.21% ≒ **10.4%**（13.0% から短縮）。
   - σ20＝2.5% の銘柄でちょうど +5%（floor）に一致。σがそれ未満なら floor の5%を死守。
2. **TBK-0010 のその他の決定（生成方式・出力スキーマ `data/volatility.json`・ブラウザ合成・
   終値ベース/表示専用・劣化動作・読み手）はすべて継承する**。本 ADR は `k` の値のみを上書きする。
3. **値の出所と整合**: `k` の正は `tools/gen_volatility.py` の `TP_K`。生成される
   `data/volatility.json` の `k` フィールドに載り、表示側（`js/selltarget.js`）はそれを参照する
   （ハードコードしない。JSON 欠損時のフォールバック既定値も 2.0 に合わせる）。
4. **以後のパラメータ（k / floor / cap / window）変更も本 ADR の改訂（新 ADR）で行う**（TBK-0010 を踏襲）。

## 📈 結果・影響（Consequences）

- 高ボラ銘柄の利確目標が一段近くなり、運用感覚に合う（例 6590: 13.0%→10.4%）。
- 低ボラ銘柄は従来どおり floor の5%に張り付くため、影響は主に高ボラ銘柄に出る。
- JSON スキーマは不変のため、読み手・テスト・データ契約の構造変更は不要（値の差し替えのみ）。
