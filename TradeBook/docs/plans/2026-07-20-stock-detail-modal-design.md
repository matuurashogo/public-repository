# 銘柄詳細モーダル 設計（チャート＋情報集約）

- 日付: 2026-07-20
- 種別: UI 機能（既存データ契約を読むだけ・新規 `data/*.json` なし → ADR ではなく設計ドキュメント／eval ティアB）
- 関連: TBK-0006（買いレベル）/ TBK-0008（場中価格）/ TBK-0010（利確目標）/ TBK-0014（支持線・抵抗線）/ TBK-0004,0005（客観指標）

## 目的

買い時ボード・保有銘柄カードの一覧では1行に収まる情報しか出せない。銘柄を選んで
**支持線・抵抗線を含む詳細（チャート＋数値）を1画面で見たい**という要望に応える。

## UX

- **起動**: 買いボード・保有カードの**行タップ → その行の下に「◯◯ の詳細を見る」ボタン**が出る
  → 押すとモーダル（下からのシート）で詳細を表示。同じ行の再タップでボタンを閉じる。
- **閉じる**: ×ボタン / オーバーレイ / Esc。
- **表示方式**: 同一画面オーバーレイ（PWA で戻るが自然・実装が軽い）。

## 内容（両方フル）

1. ヘッダー: 銘柄名＋コード＋現在値（場中が新鮮なら場中／無ければ終値）
2. **チャート**: 終値の折れ線（`data/indicators/<code>.json` の `c`・直近約250営業日）に
   **支持線＝緑 / 抵抗線＝赤の横線のみ**を重ねる（Chart.js 標準機能のみ・追加プラグイン不使用。
   横線は「全x点に定数yを持つ線データセット」で描画）。表示価格域に入る水準だけ描く。
3. 情報セクション（数値のみ。算術は JS 側で確定）:
   保有・含み損益（保有時）／支持線・抵抗線（現在値からの距離%）／利確目標（σ20・幅・保有なら目標価格）／
   買いレベル L1〜L6（到達・あと%）／客観指標（25日線乖離・75日線・出来高比・RSI・HV）。

## 実装（純粋関数と描画の分離）

- `js/detail.js`（純粋関数・テスト対象）: `buildChartModel(rows, sr, lastN)` と
  `buildDetailSections(input)`。既存データから view-model を組む。**計算はここ／`selltarget.js`**。
- `js/charts.js`: `renderStockChart(canvas, model)`（描画のみ）。
- `js/app.js`: 行タップ→ボタン→`openStockDetail(code)`（`indicators/<code>.json` を遅延取得）→
  モデル組み立て→描画。閉じる配線。
- `index.html` / `css/style.css`: モーダル DOM とスタイル。
- `sw.js`: キャッシュ v86→v87・`js/detail.js` 追加。

## データ制約

- チャートは監視リスト銘柄（`indicators/<code>.json` がある銘柄）のみ。**監視リスト外の保有銘柄は
  チャート無し**（注記表示）で、取れる数値だけ出す。クラッシュしない。
- 高値・安値は使わない（S/R のみチャート）ため、新しいデータ生成は不要。

## Eval（ティアB）

- **対象**: 銘柄詳細モーダル（チャート＋情報集約）
- **成功条件**: (1) 両ボードの行タップで詳細ボタンが出て押すとモーダルが開く。(2) データのある銘柄で
  終値折れ線＋S/R横線（支持=緑/抵抗=赤）が描画される。(3) 支持/抵抗・買いレベル・σ利確・客観指標・
  （保有なら）含み損益が数値表示される。(4) 監視リスト外・データ欠損でも例外なく「データなし」表示。
- **Grader**: Code = `tests/detail.test.js`（純粋関数11件）＋既存 `node --test` 全 pass。
  Human = 実機（Chromium/実PWA）で開閉・チャート・各セクションの目視。
- **反例**: テンプレート内で率・目標価格を計算している（算術はコード原則違反）／欠損銘柄で例外。
