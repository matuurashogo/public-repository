# 場中価格（intraday prices）表示の設計書

- 作成日: 2026-06-11
- ステータス: 設計確定（ユーザー承認済みの検討結果に基づく）/ 実装は Go 待ち
- 関連: TradeBook（表示）、買い時ボード（TBK-0006）、update-prices.yml（日次パイプライン）

---

## 1. 背景と目的

保有銘柄の含み損益と買い時ボードの「現在値」は J-Quants の日足終値（夜22時確定）のみで、
場中の値動きが見えない。場中のおおよその現在値（約20分遅延）を表示できるようにする。

### 検討の経緯（2026-06-11 調査）

| 候補 | 結果 |
|---|---|
| yfinance | ❌ GitHub Actions の共有IPでは 429 で全滅（実地報告複数） |
| stooq.com | ❌ 2026年6月初旬に Bot 対策導入で API キー発行廃止・実質終了 |
| **Yahoo チャートAPI 直叩き** | ✅ 現時点で Actions から動作（2026-06-10 付の実地報告） |
| GOOGLEFINANCE（Sheets API） | ○ 保険候補。既存のサービスアカウントを流用可。値が返らない癖あり |
| J-Quants | 場中データはどのプランにも無い |
| 証券会社API | ユーザー方針により除外 |
| スクレイピング | 規約・脆弱性により不採用 |

**教訓**: この領域の無料ソースは数ヶ月単位で死ぬ（stooq が実例）。
よって「ソースを差し替え可能にする + 壊れても安全に劣化する」ことを設計の中心に置く。

## 2. 設計原則（ユーザー合意済み）

1. **表示専用**: 場中価格は保有銘柄の含み損益と現在値表示にだけ使う。
   🟢到達判定・LINE 通知・ExitLab の検証は**終値ベースを堅持**（検証済みの前提を崩さない）
2. **契約は汎用、仕組みは最小**: 出力 JSON の契約（TBK-0008）だけ固定し、読み手の増加に備える。
   分足の蓄積・全銘柄対応はしない（最新値1点・監視リスト銘柄のみ・履歴は捨てる）
3. **ソース抽象化**: 取得部を source 単位で差し替え可能にする。Phase 1 は Yahoo のみ実装、
   GOOGLEFINANCE フォールバックはソースが死んだ時に追加（YAGNI）
4. **鮮度の明示**: 取得時刻（asOf）を必ず表示し、古いデータでの誤判断を防ぐ

## 3. アーキテクチャ

```
GitHub Actions（場中 30分ごと・平日 9:00〜15:30 JST）
   ↓ tools/fetch_intraday_prices.py
   ↓   対象: data/indicators_universe.json の銘柄（監視リスト ∪ 売買銘柄）
   ↓   ソース: Yahoo チャートAPI（約20分遅延・UA付き・銘柄ごと1リクエスト+間隔）
intraday ブランチに force-push（orphan・履歴は常に1コミット）
   ↓ raw.githubusercontent.com/.../intraday/data/intraday_prices.json
TradeBook が fetch（CORS可・約5分CDNキャッシュ）
   ├─ 保有銘柄カード: 含み損益を場中価格で表示（「13:30時点」併記）
   └─ 買い時ボード: 現在値列のみ場中表示（🟢/🟡の判定は終値のまま・注記明示）
```

### 置き場所を専用ブランチにする理由

30分ごとに main へコミットすると年3,000コミット超のノイズになり、Pages の再デプロイも
無駄に走る。orphan ブランチへの force-push なら履歴は常に1件で、main は汚れない。

## 4. データ契約（TBK-0008 として実装時に正式化）

```json
{
  "asOf": "2026-06-11T13:30:00+09:00",
  "source": "yahoo_chart",
  "prices": { "5016": 3392.0, "6855": 6950.0 }
}
```

- `asOf` = 取得完了時刻（JST、ISO 8601）。読み手は鮮度判定にこれだけを使う
- `prices` = 4桁コード → 最新価格（遅延約20分）。取得失敗した銘柄はキー自体を出さない
- **読み手の劣化動作**: ファイルが取れない・asOf が古い（>90分）場合は従来どおり
  日足終値の表示へフォールバックする（エラーにしない）

## 6. リスクと割り切り

- **Yahoo も非公式**: いつ止まってもおかしくない。止まった場合は表示が終値に自動フォールバック
  するだけで実害なし（判定系は元々終値）。GOOGLEFINANCE ソースの追加はその時に行う
- **Actions cron の遅延**: 定刻から数分〜15分ずれることがある。asOf 表示で吸収
- **祝日**: 休場日も空振り実行される（取得値が前日終値のまま=無害。コミット差分なしならpushしない）
- **昼休み（11:30〜12:30）**: 値が動かないだけで無害
