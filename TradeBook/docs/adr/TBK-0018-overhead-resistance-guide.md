## ❓ コンテキスト（背景と課題）

TBK-0016 の利確目標は「min(σ連動目標, 取得単価より上の最寄り抵抗線)」で、
**利確候補を取得単価より上の抵抗線に限定**している（含み損時に損失確定を"利確"と
表示しない安全弁）。しかしこの制約により、**含み損ポジションでは全抵抗線が候補から
外れ、σ連動目標だけが残って現在値から遠い数字になる**（実例 6855: 現在 5,960・
取得 6,741 で目標 7,623・あと +27.9%）。ユーザーから「これでは使えない」と指摘。

一方、現在値のすぐ上にある抵抗線（6855 なら 6,080・あと +2%）は、含み損でも
「戻ったらまずここでつかえそう」という**実用的な戻りメド**であり、これを見たい。

## 💡 決定事項（Decision）

1. **「上値メド」を新設**（`detail.buildDetailSections` の `overhead`）: **現在値より上の
   最寄り抵抗線**を、取得単価に関係なく常に算出する。
   ```
   上値メド = min( sr.resistance のうち 現在値 < 水準 )
   dist = (上値メド − 現在値) / 現在値
   ```
   現在値より上に抵抗線が無ければ null（非表示）。
2. **利確目標とは別物として併記**: 利確目標（TBK-0016・取得単価基準のプラス目標）は
   そのまま維持し、**上値メドを詳細モーダルの「利確目標・上値メド」ブロックに追加**する。
   これにより「利確＝利益目標」の意味は守りつつ、含み損でも実用的な戻りメドが見える。
3. **表示範囲**: 詳細モーダルのみ（保有カード・買い時ボードは列を増やさない）。
   sr_levels 未配信・現在値なしの銘柄は非表示（劣化動作）。

## 📈 結果・影響（Consequences）

- 含み損ポジションでも「直上の抵抗線＝現実的な戻りメド」が一目で分かる
  （6855: 利確目標 7,623 は遠いが、上値メド 6,080 あと +2% が実用値）。
- 上値メドは表示専用・現在値基準で、TBK-0016 の利確目標ロジックは不変（後方互換）。
- 反発率（TBK-0017）と併せて見れば「直上の抵抗線が過去どれだけ効いたか」も分かる。

  (3) dist が (水準−現在値)/現在値、(4) 利確目標ロジックに影響しない
