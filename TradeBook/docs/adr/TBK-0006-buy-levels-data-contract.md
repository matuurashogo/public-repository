## ❓ コンテキスト（背景と課題）

現在の投資フローは「AlphaScorer で良い銘柄を選ぶ → 監視リストに入れる → テクニカル目線で
安くなったら買う」だが、最後の「安くなったら」が数値基準のない裁量（チャートの節目・雰囲気）に
なっており、(1) あといくら下がったら買いかが事前に見えない、(2) 自分の「節目感覚」を後から
検証できない、という課題がある。

そこで監視リスト銘柄ごとに「買いレベル」（節目の数値代理・複数併記）を日次計算し、
アプリのボード表示と LINE 通知の両方から参照できる静的 JSON として同梱する。
設計の全体像は `docs/plans/2026-06-10-buy-levels-board-design.md` を参照。

## 💡 決定事項（Decision）

1. **生成方式は TBK-0002 の流儀を踏襲する**: 監視リスト（`data/indicators_universe.json`）の
   銘柄のみを対象に、`tools/gen_buy_levels.py` が jquants-data の日次株価から事前計算し、
   GitHub Actions（update-prices.yml）で日次更新・コミットする。
2. **レベル定義（6本・調整後終値ベース）**:

   | ID | レベル価格 | 意図 |
   |---|---|---|
   | L1 | MA25（25日移動平均） | 定番の押し目 |
   | L2 | MA25 × 0.95 | やや深い押し |
   | L3 | MA25 × 0.92 | 深い凹み（VolDipSignals reversal と同水準） |
   | L4 | 直近20日最高終値 × 0.90 | 高値からの押し率（浅め） |
   | L5 | 直近20日最高終値 × 0.85 | 高値からの押し率（深め） |
   | L6 | 直近60日最安終値 | サポートライン（下値の節目）の代理 |

   どのレベルが正解かは決めつけず**全レベルを併記**する。実際の買い記録との突合
   （較正・Phase 2）で主基準を絞っていく。
3. **出力スキーマ `data/buy_levels.json`**（1ファイル・監視リスト全銘柄・コード昇順）:
   ```json
   {
     "updated": "2026-06-10",
     "source": "jquants-data prices (調整後終値ベース)",
     "near_threshold": 0.03,
     "stocks": [
       {
         "code": "6855",
         "close": 6950.0,
         "rebound": false,
         "levels": [
           { "id": "L1", "label": "25日線", "price": 7430.0, "dist": 0.0691, "hit": true }
         ]
       }
     ]
   }
   ```
   - `close` = 最新の調整後終値（小数1桁）
   - `rebound` = 陽転フラグ（最新終値 > 前日終値。下げ止まり確認）
   - `price` = レベル価格（小数1桁） / `dist` = (レベル価格 − 現在値) / 現在値（小数4桁。
     負 = レベルは現在値より下 = 「あと |dist| 下がったら到達」）
   - `hit` = 現在値 ≤ レベル価格（到達済み）
   - `near_threshold` = 「接近」表示の閾値（`dist` が −near_threshold 以上かつ未到達）。
     表示側・通知側はこの値を参照し、ハードコードしない
4. **読み手は2つ**: アプリの「買い時ボード」カード（`js/buylevels.js`）と、private 側の
   LINE 通知スクリプト。どちらも本契約の JSON のみを入力とし、独自に再計算しない。
5. **劣化動作**: JSON が無い・取得失敗時はカードを非表示にする（致命的にしない）。
   履歴が不足する銘柄（上場直後など）は計算可能なレベルのみ出力し、不足レベルは省略する。

## 📈 結果・影響（Consequences）

- 監視リスト銘柄の「あといくら下がったら買いか」が毎日自動で数値化され、指値の目安になる。
- レベル定義の変更（本数・倍率）はスキーマ変更を伴うため、本 ADR の改訂（新 ADR）で行う。
- ファイルは監視リスト銘柄のみ・1ファイルのため、git 履歴への影響は軽微（indicators と同方式）。
